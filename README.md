## 📜 소스 관리 룰

### 브랜치 전략

- **Prefix**
    - daangn4jo : 최종 병합 브랜치
    - release : 배포 브랜치
    - dev-ios
    - dev-fe
    - dev-be
- **Infix**
    - feature/
    - fix/
    - refactor/
- **Suffix**
    - 이슈번호 + 기능 명
    - Github issue의 ’Create branch’ 활용 권장
- **예제**
    - feature / ios / 12-상품목록-UI-구성
